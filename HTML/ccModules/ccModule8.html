Module 8: User Input

This module is about the ways in which we, as humans, can provide input for web scripts to use with an algorithm, to produce output. I have come up with 4 different ways we can provide input to web technology.

Using the mouse

Using the mouse, also including clicking the mouse and tapping screens for mobile technologies, are by far the most common option for user input. The mouses is useful because it can be perfectly controlled and is always precise in its movements. Many things can be created using proximity to mouse, hover effects, moving direction, coordinates, or clicking actions.

Using the keyboard

Using the keyboard is another common user input method. Creating things according to what letter a user presses, or combinations are common. Additionally, many things can be created using words, and type that the keyboard can help the user create.

Using facial expressions from webcam

Facial expressions are something I have not seen used in many web scripts at the moment. There is definitely concern about safety and recognision of faces, which may hinder this progression, however there are some really cool things that can come from both facial expressions, and face layouts too.

Using sound from microphone

Using sound or audio recorded can be a really cool way of integrating user input. This immediately makes me think of Scream Go Hero, and old mobile game similar to 'Flappy Bird' where you had to scream to jump. Singing, or other music could also be interesting for input.

Poses from webcam
